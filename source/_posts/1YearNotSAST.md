转眼间, 距离 {% post_link 2YearsSAST "因为科协的事情破防 emo" %} 已经快一年的时间了. 我们的下一届也快完成使命, 要寻找下一届的主席了. 这一年里... 说来可笑, 我的感觉是开发的工作还是老登和 [落选主席 wst](https://blog.xiaoaojianghu.fun/) 在干, 我把运维的重任丢给了下一届副主席, 好在没有太让我失望 (其实也没什么运维的活) (其实说要干的饼也没有人推动, 都停留在饼上). 马上又要提名新一任主席了, 我倒是想看看下一届会是什么样子.

## 网络部三件套

这一年, 网络部经典的事情还在尝试推进, 但是感觉并不顺利.

基础技能培训似乎倒闭了, 不知道. 只是听说. 但是我既没有看到公众号推送, 也没有看到群, 也没有看到有人讨论. 大约的确是似了.

[暑期技能培训](https://summer25.net9.org/) 改成了线上放视频, 没有直播了. 内容差不多, 毕竟基础的哪些东西也就那些, 翻来覆去讲了很多年了, 也玩不出什么花样. 当年 (前年) 我讲 Git 和 Django 的时候还是基于我当时的一些经验和理解, 也不知道去年的讲师水平如何. (哦, 是我的下一任讲的, 那应该还行吧x). 不过那时候, Vibe Coding 应该已经成为很重要的一部分了. 如果是我, 我一定会要求培训里面加入相关内容的.

Weekly9 变成 Seasonly9 了, 总共就发了两篇...? 一篇整活文章, 讲 [如何在华为平板上面装带 GUI 环境的 Linux 发行版容器](https://mp.weixin.qq.com/s/EtqHDlK3HSmj3fzYrJc6bQ), 这篇文章是 wst 写的, 用于演示的机器是我的平板; 另一篇是 [介绍神经网络的](https://mp.weixin.qq.com/s/8Aec7NTgBQ95QAL8XFAN5A). 这之后就已经是这学期了. 确实 Weakly.

## 别的

然后网络部的事情无非就是三大类, 整活, 接活, 救火.

整活的话... 好像只有我会整活 (x)

接活... 这一年最大的活应该是系里要整一个希望 "拉齐同学们的 AI Access 能力, 提供免费的 SOTA 模型" 的系统. 一开始分给小朋友开发, 让我带头; 我当时眼看着如果我真带头的话, 后面大概率就只有我开发了, 小朋友们就不干了, 因此我在指导了一下技术选型之后我就跑了. 后来比较抽象的是, 我们的落选主席 (对, 又是他) 自告奋勇来 lead 开发, 于是... 后来看上去小朋友们也没怎么干活, 主要就是他在指挥他的 Copilot 干活. 经过了大半年 (?), 这个系统的功能实际上已经挺完整了, 系里提供了 qwen API, 系统上面有对话, wst 还接入了 PDF 识别等等一堆东西, 还做了 Proxy API. 最近 OpenClaw 很火, 我用这个 API 当 OpenClaw 的 API 用, 感觉还不错 x ~~(毕竟是白嫖)~~ 但是至今还没有正式宣发, 只是上线了.

救火呢, 这一年说实话也没什么特别大的火. 毕竟这些服务, 你不动它, 一般也不会坏. 倒是确实有一些火, 一个是 {% post_link DellMemoryMirrorMode "Zeus 的内存坏了一根" %}, 正好是暑假时间; 一个是快寒假的时候 {% post_link "CheckSMARTESXiDell13thGenServer" "Hera 的 RAID5 炸了一块" %}, 我敦促他们薅了两块 4T 盘换上去了. 别的无非就是掉准入了啥的, 都是小问题. 但是, 我怎么感觉这些事情都还是我或者 wst 先发现, 然后再通知这一届的同学处理呢? 也只有我愿意研究和折腾.

## Callback

还有一些 Callback. 去年我在 emo 的时候说,

> *如果服务没人维护, 用户自然会找到出路.*

这句话确实是至理名言罢... 邮件服务没修, 反正确实是没人管了, 没有邮件也不是不能用. 然后之前隔三差五问我什么计金的同学, 信技的同学没有酒井 ID 能不能修, 我都不由分说往行政上面推, 要他们赶紧找学校申请 SSO 权限, 到头来也都不管了. 然后有人当时问我想查当年的同学资料说密码登不上酒井 ID, 也先往弱密码, 安全, 行政, 信息办头上推, 最后也不了了之 (不过我后来帮他查了一下, 发现数据似乎都已经丢失了, 系统里没人了).

去年我说,

> 是否我将成为一枚筹码, 抑或日后运维的工具

现在看来, 是也不是. 似乎依然只有我能运维, 因此出了事情确实有一部分人会找我. 但是呢, 我的态度可以说发生了变化. 毕竟我已经不在岗位上, 没有搞运维的 *职责* 了. 取决于谁找我, 如果是朋友, 我搭把手, 自然是没有问题; 但是如果是一口官腔, 想来 Push 我干活, 那么我踢皮球, 打太极, 他也没法拿我怎样. 至于我当时 {% post_link OpenProxyScanner 自己折腾的灵车 %}... 我不得不说, 它还真不是很灵, 在一年里面几乎没有出现问题.

## 管理

有人可能要问了, 这一年, 你没有什么想喷的吗? 这篇文章火药味似乎不是很足啊?

我只能说, 网络部确实有些萧条, 但是没有出什么大锅, 在重大的事情上处理的也没那么糟, 至少我没有什么能喷的. 更大的锅明显在管理上.

(写到这里, 我怎么感觉我血压都高了... 为了我的人身安全和身体健康, 我得歇一会)

网络部确实是没有他妈的出锅, 但是整个科协我觉得可以说是出了很大的锅.

最大的锅就是他妈的主席, 到底是谁觉得 xry 很能当科协主席, 要把他拉去选举的? 到底是他妈的他想当科协主席带好队, 还是想他妈的给他一个科协主席的头衔去评特奖? 无论到底是哪个, 我认为, 能看到的结果都是, 他当科协主席之后没有尽到主席的职责, 没有带好队, 把自己的政绩工程放在第一位, 至于其它的 -- 无论叫 "祖传" 也好, "传统" 也罢 -- 的事情压根不关心, 不上心, 项目出问题不知道嘘寒问暖, 工期有所拖延不知道隔两天催一下, 快要交付了不检查, 报销的事情似乎也不甚关心, 上下沟通常常卡在这里 (听说的). 甚至 wst 跟我说他都不想找 xry, 什么事情都直接跟 xxh, jyj 甚至 hwt 老师对接. 这是一个非常大的问题 -- 那我他妈的要主席干什么? 我要他拿特奖吗? 还是科协需要一个什么精神领袖带领大家走向 *赢!* ?

这就要提到我为什么当时不去竞选科协主席. 当然, 我去年的吐槽博客里面写了一些, 我让 AI 总结了一下, 大多都是感性的, 包括;

- 运维工作压力大, 里外不是人
- 在卷绩点的环境下, 运维工作难以获得认可
- 找不到接班人, 传承困境
- 服务存在太多隐患, 担心出问题自己要背锅

当然这些都是事实. 但是实话说这些都能克服. 真正我觉得我不适合 (当然, yjgg 也这么觉得我不适合) 的原因是, 我一直不是一个能很好带队的人, 至少我自己这么觉得. 我可以做技术骨干, 但是在催别人干活和鼓励大家一起干活这件事情上, 我个人的评价一直是, 爱干不干, 不干拉倒.

对于我来说, *催别人干活比我自己干活累*, 因为 a) 如果他真的没兴趣干活, 我催感觉像在强迫劳动; b) 他如果确实是没时间, 催了也没用; c) 让别人干活, 结果不一定能达到我的标准, 还不如我自己干了算了; d) 否则我得花很大的力气在向他传递为什么我决定要这么做上面, 除非我们两个人在行业上已经有比较多的经验, 能够几句话把事情讲清楚.

这样一来, 对于整个网络部的事情, 如果我是主席, 大概率所有活 / 几乎所有活都是我自己干, 副主席得不到培养, 更别提小登了. 这对于整体的团队建设是不利的. 而另外一个问题是, 如果我是主席, 那么其他人会觉得 a) 有个靠山 / 垫背的; b) 反正如果达不到我的标准, 那么最终还是我干或者要被我骂; c) 如果他们拖着最后就不用干了; d) 他们大概知道我是什么性子, 我在我确实认为会降低他们的积极性.

反倒是如果我不当主席成为老人, 那么最后背锅的不是我, 我可以更多站在一个指导者的角色上教他们干活, 让他们自己被 push, 给他们更多自由裁量的空间, 把决定权交给他们.

所以说白了, 科协主席这个位置应该交给一个... 能力没那么强, 但是能带队的人. yjgg 自称是这样的人 (虽然我觉得能力还是很强的), 我觉得他基本做到了. 很可惜的是, 去年似乎没有这样的人... 吗? wst 其实算半个, 只是他平常太随性, 以至于没那么让人觉得他能带队.

但是现在并非这样. 今年, 说实话, 我觉得科协处于一个半停摆的状态.

最近的智能体大赛出了巨大的锅, 他们上传的文档和代码不是一年的东西... 选手对着 2022 年的 AntWar 的重置版的文档, 发现下载下来的代码是 2025 年 Generals 的, 而且只更改了一个 int 参数. 有选手发现问题之后在群里面喷人, 智能体部的人居然没有一个知道怎么回事. 对于一个如此抽象的错误 (先不说 in the first place 这种错误是怎么产生的) 没有最起码的应急处理措施, 还要找当年开发平台和游戏的老学长来解决. 当然, 更好笑的还在后面, 这件事上报到主席之后, 他的第一反应是给几个人拉个小群, 要求他们 "每日汇报问题解决进度", 还点名了一个老学长...

亏得这老学长脾气没那么暴躁, 只是在群里表示 "我就不参与了". 要是我敢因为这抽象事点我的名, 我少说在群里面当面把每个人骂一通. 简直是了不得了, 开发的时候搞不明白, 测试的时候搞不明白, 出了问题搞不明白, 还要指着老学长的鼻子让他汇报进度? 你当你是谁啊?

还有, 这玩意难道还需要几天来解决吗? 难道不应该是当场拉会, 不解决掉不让睡觉吗...?

还有... 他妈的, 测试呢?

**由于管理者的失位**, 我觉得科协的相当多的组件都处于停摆的状态. 从我听到的, 老人们的评价, 我自己的观察, 上一届的其他副主席的评价, 以至于下一届的评价来看, 这一年, 秩序基本是混乱的, 事情是没有人推动的, 那么结果是显然的: **倒闭**.

## 去路何在

在这样的情况下, 我个人至少是不愿意再为科协投入更多的精力了. 这当然包括我当时整活的部分成果. 当然... 我知道如果老人们都这么想, 科协真的就完蛋了. (参见 Harry 的朋友圈...?)

又快要到了选举的时间, 下一届会是什么样子呢? 还会有 9 个副主席的人选吗? 现在的 9 位副主席看到现在的状况, 还会有人想接手这个烂摊子当主席吗? 或者还得让导员指派一个?

哦还有, 今年宿舍要装修, 科协活动室也要搬迁了, 搬到哪里去还不一定. 还有科协的固定资产, 希望别被橄榄.

期待一个后续.
